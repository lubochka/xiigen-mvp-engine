# E2E Snapshot Fix Plan

**Date:** 2026-04-15
**Issue:** Root cause analysis identified two compounding regressions that caused missing/misplaced UI snapshots across flows 05+.
**Scope:** Normalize existing flows (01, 03, 04, 05, 06) + add screenshot captures to all flows 07+ with client pages.

---

## Root Cause Recap

1. **Plan language drift at FLOW-03:** IMPL-STATE.json labels changed from `"Playwright + UI snapshots"` (FLOW-01/02) to `"React pages + Playwright"` (FLOW-03+). The word "snapshots" dropped from planning signal.
2. **Template inertia carried snapshots through FLOW-06** via copy-paste from user-registration.spec.ts, then exhausted at FLOW-07 when new test patterns were written from scratch.
3. **Naming convention violation (Rule 16):** FLOW-01/03/04/05/06 tests write to `docs/e2e-snapshots/flow-NN/` (numeric) instead of `docs/e2e-snapshots/{slug}/` (semantic). Only FLOW-02 (profile-enrichment.spec.ts) uses the correct slug pattern.

---

## Target State

Every client e2e spec file follows this canonical pattern (from `profile-enrichment.spec.ts`):

```typescript
const SNAP = (name: string) =>
  path.join(__dirname, '..', '..', 'docs', 'e2e-snapshots', '{semantic-slug}', name);

// Then in each test:
await page.screenshot({ path: SNAP('01-description.png'), fullPage: true });
```

Where `{semantic-slug}` matches the flow directory slug from CLAUDE.md (user-registration, event-management, marketplace-payments, etc.) — NEVER `flow-NN`.

---

## Execution Strategy

**Sequential, flow-by-flow. Not parallel.**
After each flow:
1. Update the test file (path normalization + screenshot additions)
2. Update `docs/snapshot-fix-state.json` marking the flow COMPLETE
3. Commit with message `fix(e2e-snapshots): FLOW-NN {slug}`
4. Push to origin

Tests will NOT be executed in this session (no Vite/Playwright runner available). The PNGs will be generated by the user's CI or next manual test run.

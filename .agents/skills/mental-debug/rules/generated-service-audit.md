# Rule 12: Generated Service Audit (adapted from code-gen-output-audit)

## What It Catches

Generated `.ts` files (output of AF-1 Genesis or AF-7 RAG Update) that contain DNA violations. Generated code has no IDE warnings and no TypeScript compile errors for DNA violations — it compiles and runs but produces wrong behavior.

## The Problem

When AF-1 generates a TypeScript service file, it can produce code that violates DNA patterns (DNA-1 through DNA-9). The violation is not caught by:
- TypeScript compiler (compiles fine)
- ESLint (no lint rule for DNA patterns)
- Unit tests (tests the behavior, not the pattern)

The violation is only caught by manually running the DNA compliance checklist against the generated output.

## When to Run This Audit

**Mandatory:** Before committing any file generated by AF-1 Genesis or AF-7 RAG Update.

**Triggers:**
- `af1-genesis.ts` produces a new service file
- `af7-rag-update.ts` regenerates an existing service file
- Self-developing engine updates any `.ts` file in `server/src/`

## DNA-1 Through DNA-9 Checklist for Generated Files

```
DNA-1: No entity-specific model classes
  VIOLATION: class OrderModel { ... }  in generated service
  DETECT:    grep "class [A-Z].*{" <generated-file.ts>
  FIX:       Replace with generic Record<string, unknown> or T extends BaseModel

DNA-2: No hardcoded field selectors
  VIOLATION: .find({ orderId: input.orderId })  in generated service
  DETECT:    grep "\.find({" <generated-file.ts>
  FIX:       Use BuildSearchFilter(input) — let factory define field mapping

DNA-3: No business logic in error handlers
  VIOLATION: catch (e) { if (e.type === 'PAYMENT') { ... } }
  DETECT:    grep "catch" <generated-file.ts> — inspect each catch body
  FIX:       catch → return DataProcessResult.failure(e.message) only

DNA-4: Service extends MicroserviceBase
  VIOLATION: class GeneratedService { ... }  — no extends
  DETECT:    grep "class.*Service" <generated-file.ts> — check for "extends"
  FIX:       class GeneratedService extends MicroserviceBase { ... }

DNA-5: tenantId from context, not parameter
  VIOLATION: async processOrder(tenantId: string, ...)
  DETECT:    grep "tenantId" <generated-file.ts> — check if it's a param
  FIX:       Remove tenantId param — read from this.cls.get('tenantId')

DNA-6: No entity-specific controllers
  VIOLATION: @Controller('orders')  in generated file
  DETECT:    grep "@Controller(" <generated-file.ts>
  FIX:       Remove controller — engine's DynamicController handles routing

DNA-7: Event subscriptions have dedup ID
  VIOLATION: @Subscribe('order.completed') without dedup field
  DETECT:    grep "@Subscribe(" <generated-file.ts> — check dedup
  FIX:       Add dedupId field to CloudEvents envelope

DNA-8: Document stored before queued
  VIOLATION: await this.queue.enqueue(event) before storeDocument()
  DETECT:    Look for enqueue() calls — is storeDocument() called first?
  FIX:       Reorder: storeDocument() → enqueue()

DNA-9: CloudEvents wrapper
  VIOLATION: await this.queue.enqueue({ orderId, type: 'completed' })
  DETECT:    grep "enqueue(" <generated-file.ts> — check for CloudEvents wrapper
  FIX:       Wrap in CloudEvents: { specversion, type, source, data: payload }
```

## Audit Script (run on any generated file)

```bash
FILE="server/src/generated/example-service.ts"

echo "=== DNA-1: entity-specific models ==="
grep "class [A-Z].*{" $FILE

echo "=== DNA-2: hardcoded field selectors ==="
grep "\.find({" $FILE

echo "=== DNA-3: business logic in catch ==="
grep -A5 "catch" $FILE | grep -v "DataProcessResult.failure\|logger\|return"

echo "=== DNA-4: extends MicroserviceBase ==="
grep "class.*Service" $FILE | grep -v "extends MicroserviceBase"

echo "=== DNA-5: tenantId as parameter ==="
grep "tenantId:" $FILE | grep "string\|UUID"

echo "=== DNA-6: entity controllers ==="
grep "@Controller(" $FILE

echo "=== DNA-9: CloudEvents wrapper ==="
grep "enqueue(" $FILE
```

Expected output for a clean file: all sections empty.

## Anti-Pattern

"The generated file compiles so it must be correct." Compilation does not verify DNA compliance. Run the audit script before any commit of generated code.
